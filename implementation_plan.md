# Implementation Plan

[Overview]
Исправление неработающего функционала на странице товара: выбор цвета товара, кнопки "Стоимость доставки" и "Рассрочка без переплаты".

В текущей реализации на странице товара присутствуют UI элементы, но отсутствует логика обработки взаимодействия пользователя с ними. Проблемы касаются трех основных компонентов: выбор цвета, кнопка стоимости доставки и кнопка рассрочки. Все элементы визуально отображаются, но не реагируют на клики пользователя.

[Types]  
Добавление реактивных переменных и обработчиков событий для интерактивных элементов.

Требуется добавить:
- `selectedColor` - реактивная переменная типа Ref<number> для хранения индекса выбранного цвета
- Массив доступных цветов товара с названием и кодом цвета
- Методы обработки кликов для каждой кнопки
- Модальные окна или логика перехода при нажатии на кнопки

[Files]
Изменения будут внесены только в файл страницы товара.

Детальный breakdown:
- Изменить существующий файл `pages/product/[id].vue`
  - Добавить реактивное состояние для выбранного цвета
  - Добавить обработчик клика для выбора цвета
  - Добавить обработчик клика для кнопки "Стоимость доставки"
  - Добавить обработчик клика для кнопки "Рассрочка без переплаты"
  - Добавить визуальное выделение выбранного цвета
- Дополнительно при необходимости будут созданы компоненты модальных окон

[Functions]
Добавление новых функций-обработчиков событий.

Детальный breakdown:
- Новая функция `selectColor(index: number)` - для обработки выбора цвета
- Новая функция `openDeliveryInfo()` - для открытия информации о доставке
- Новая функция `openInstallmentInfo()` - для открытия информации о рассрочке
- Все функции будут расположены в секции script setup файла `pages/product/[id].vue`

[Classes]
Изменения в CSS классах для визуального отображения выбранного состояния.

Детальный breakdown:
- Добавление условных CSS классов для элементов выбора цвета
- При выборе цвета добавляется класс `border-primary ring-2 ring-primary ring-offset-2`
- Для невыбранных цветов остаются стандартные классы
- Никакие новые классы не создаются, используются существующие Tailwind классы

[Dependencies]
Новые зависимости не требуются.

Все изменения будут реализованы с использованием уже подключенных библиотек: Vue 3, Nuxt 3 и Tailwind CSS. Дополнительные пакеты устанавливать не нужно.

[Testing]
Проверка работоспособности всех интерактивных элементов.

Требуется проверить:
1. При клике на цвет он должен визуально выделяться
2. При повторном клике на другой цвет выделение должно переключаться
3. При клике на "Стоимость доставки" должно открываться соответствующее модальное окно или происходить переход
4. При клике на "Рассрочка без переплаты" должно открываться соответствующее модальное окно или происходить переход
5. Все элементы должны сохранять свое состояние при обновлении страницы

[Implementation Order]
Последовательность реализации исправлений по шагам.

1. Добавить реактивную переменную для выбранного цвета и массив доступных цветов
2. Реализовать функцию выбора цвета и подключить её к элементам UI
3. Добавить условные классы для визуального отображения выбранного цвета
4. Реализовать функцию обработки клика по кнопке "Стоимость доставки"
5. Реализовать функцию обработки клика по кнопке "Рассрочка без переплаты"
6. Протестировать все элементы на работоспособность
7. Проверить визуальное соответствие и отзывчивость интерфейса
