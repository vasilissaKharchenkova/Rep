# Implementation Plan

[Overview]
Адаптировать сайт CLICKWOOD для мобильных устройств: изменить шапку (бургер-меню, логотип по центру), убрать изображение из секции подписки на главной, переработать слайдер изображений в карточке товара.

Проект построен на Nuxt 4 + Vue 3 + Tailwind CSS. Требуется модифицировать три файла: `components/Header.vue` (навигация и функциональные кнопки скрываются в бургер на <768px), `pages/index.vue` (убирается изображение стула в секции подписки на мобильных), `pages/product/[id].vue` (слайдер изображений — без колонки миниатюр, с кнопками по бокам и индикатором прогресса на мобильных). Изменения затрагивают только шаблоны (template) и стили; логика (script) практически не меняется, только добавляется состояние для бургер-меню.

[Types]
Типы не изменяются — все необходимые типы уже определены.

Никакие типы, интерфейсы или перечисления не добавляются, не изменяются и не удаляются.

[Files]
Изменяются три существующих файла, новые файлы не создаются.

- `components/Header.vue`:
  - Добавить ref `isMobileMenuOpen` (boolean, по умолчанию false).
  - В шаблоне: на экранах < md скрыть навигационные ссылки (Каталог, Коллекции, О нас, Доставка) и функциональные иконки (Поиск, Корзина, Аккаунт).
  - Добавить кнопку бургер-меню (три горизонтальные линии), видимую только на < md.
  - Добавить выезжающее/выпадающее мобильное меню со всеми ссылками и функциональными иконками внутри.
  - Логотип CLICKWOOD центрировать на мобильных (уже есть, но проверить выравнивание).
  - На >= md оставить всё без изменений.

- `pages/index.vue`:
  - В секции `.newsletter-section` (строка 184-209) добавить условие: изображение стула (`<img src="/images/4df9807e3d419d1c0d0bd39c60a19f...">`) скрывается на экранах < md с помощью Tailwind-класса `hidden md:block`.

- `pages/product/[id].vue`:
  - В секции изображений (строки 161-189), внутри `flex gap-4`:
    - Колонка миниатюр (строка 162-169) скрывается на < md: добавить класс `hidden md:flex`.
    - Главное изображение (строка 172-188) остаётся, но на < md кнопки навигации должны быть более заметными (крупнее, с затемнённым фоном).
    - Снизу от главного изображения добавить индикатор прогресса (ряд точек или полоску), показывающий `activeImage + 1` из `currentVariantImages.length`. Показывать только на < md.
  - Вариант: на < md также добавить отступы/поля, чтобы изображение не упиралось в края.

[Functions]
Изменяемые функции не требуют новой логики — только условное отображение в шаблоне.

- `Header.vue`:
  - Новая функция: `toggleMobileMenu()` — переключает `isMobileMenuOpen`.
  - Новая функция: `closeMobileMenu()` — устанавливает `isMobileMenuOpen` в false (вызывается при переходе по ссылке).
  - Функция `handleClickOutside` (существующая) — не изменяется.

- `pages/product/[id].vue`:
  - Функции `nextImage` и `prevImage` (строки 111-121) — не изменяются.
  - Функции `onTouchStart`, `onTouchEnd` (строки 123-136) — не изменяются.
  - Все функции и вычисляемые свойства остаются без изменений, изменяется только шаблон.

[Classes]
Классы не изменяются — проект не использует пользовательские CSS-классы, только Tailwind utility-классы.

[Testing]
Ручное тестирование на мобильных устройствах и в инструментах разработчика (Chrome DevTools device mode).

- Проверить Header:
  - На экране < 768px: логотип по центру, бургер-иконка видна, меню открывается/закрывается, ссылки работают.
  - На экране >= 768px: всё как было (навигационные ссылки видны, поиск, корзина, аккаунт).
- Проверить главную страницу:
  - На мобильном изображение стула не отображается, текст и форма подписки отображаются корректно.
- Проверить карточку товара:
  - На мобильном: нет колонки миниатюр, кнопки навигации работают, индикатор прогресса показывает текущую позицию.
  - На десктопе: колонка миниатюр отображается как раньше.

[Implementation Order]
Изменения независимы и могут выполняться в любом порядке, но рекомендуется: Header → Главная (убрать изображение) → Карточка товара (слайдер).

1. Модифицировать `components/Header.vue` — добавить состояние бургер-меню, кнопку-бургер, мобильное меню. Скрыть навигацию и функциональные кнопки на < md.
2. Модифицировать `pages/index.vue` — скрыть изображение стула на < md с помощью Tailwind-класса `hidden md:block`.
3. Модифицировать `pages/product/[id].vue` — скрыть колонку миниатюр на < md, изменить стилизацию кнопок навигации, добавить индикатор прогресса под изображением.
